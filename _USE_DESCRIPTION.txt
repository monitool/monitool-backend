This is monitool full backend: API + DB.
Loopback is very easy to use and allowed me to quickly create whole backend for monitool.
API changed a bit because loopback has its own ways to CRUD data.
I have no idea (and time) how to generate static HTML page with API documentation.
When you run app, you will see whole API doc under localhost:3000/explorer/

Whole code was mainly generated by loopback framework.

// ========================================================================================================= //

How to run the project?

Istall node.js
Install loopback framework http://docs.strongloop.com/display/public/LB/Installing+StrongLoop
Install MongoDB and run it on default port.
In project directory enter command: "node ."
To see API explorer go to http://localhost:3000/explorer/ you can CRUD elements there
complexStat is under Data, so http://localhost:3000/api/data/complexStat ...

Ps. complexStat is generated in real time by server. To do it server use data from sensors.
You can only GET complexData, no POST, PUT or DELETE needed.

1. How to create loopback app?

1.1. Install Node.js
1.2. Install mongoDB
1.3. Install loopback: http://docs.strongloop.com/display/public/LB/Installing+StrongLoop
	on Windows use
		yo loopback
	instead of
		slc loopback
1.4. Enter: yo loopback
	enter project name
	
	Now loopback will create all the files and folders.
	
1.5. Now create data source (in new directory)
	enter yo loopback:datasource
	enter name and database for that source (monitoolDataSource). Choose mongoDB. 
	Your data source is now defined in file ./server/datasources.json
	In this file define db properties:
	"monitoolDataSource": {
		"host": "localhost",
		"port": 27017,
		"database": "monitool",
		"name": "monitoolDataSource",
		"connector": "mongodb"
	  }
	  
1.6. Create model.
	Loopback will create all REST API for you.
	ener yo loopback:model
	enter model name: Data
	model data source: source that you defined (monitoolDataSource)
	enter some properties excluding ID: 
		sensorToken type string
		date type date
		cpuLoad type numeric
		memLoad type numeric
		
	press enter to accept. Your model is defined in file ./common/models/data.json
	
	do the same thing to create model for Sensor
	
1.7. Start db
	in your monitool project create folder db
	go to the mongo istallation dir: C:\Program Files\MongoDB\Server\3.0\bin
	enter .\mongod.exe --dbpath C:\node\monitool\db
	
	your database server should start
	
1.8. Start node.js server
	in your monitool project directory enter node .
	
	in your browser enter http://localhost:3000/explorer/ - this is place with api to models
	
1.9. complexStat
	complexStat returns json file with statistics based on Data model. In my opinion the best thing to do is to add remote method for Data model.
	So open  file common/models/data.js and you can define there remote method to return complex statistics.
	
	complexStat is generated by server realtime.
